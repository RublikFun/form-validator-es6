A basic Storyboarder.

  • A Storyboarder can be any dom element.
  • A Storyboarder is the parent element to all scenes.
  • By default the Storyboarder will look for the '.js-storyboarder' class.
  • You can pass it a queried dom element to override the default.
  • The Storyboarder should filled with scenes.
  • By default the Storyboarder will auto initialze.
  • To Disable the auto initialization, pass false as the second argument
<div class="js-storyboarder">
</div>

A basic Scene.

  • Scenes can be any dom element.
  • Scenes are individual panels within a Storyboarder that can contain answers.
  • Scenes should each have a navigation button.
  • By default the storyboarder will look for the '.js-scene' class.
  • The storyboarder will make a navigation flow through the scenes as they appear in the dom.
  • The Storyboarder will add a 'js-active' and 'active' class to the scene when it is the current scene
  • The Storyboarder will remove a 'js-active' and 'active' class from the scene once the scene is valid and the user clicks the button
<div class="js-scene">
</div>

A basic Button.

  • Buttons can be any dom element.
  • Buttons are linked to the navigation.
  • They finish the scene when clicked, which prompts the storyboarder to move to the next scene.
  • By default the scene will look for the '.js-button' class.
<button class="js-button">
</button>

A basic Answer

  • An answer is an input element with a type.
  • By default, the scene will look for a '.js-answer' class.
  • Answers can be configured by data attributes.
  • The most essential attribute is the data-field.
  • The data-field is used for three things:
    • To structure the reported object
    • To link similar answers ( checkboxes )
    • To validate fields
<input class="js-answer" data-field="answer-intro">

Answer Validation

  • Invalid inputs are reported two ways:
    • On the scene
    • On the invalid input
  • Both of these elements will receive a 'js-error' and 'error' class.
  • The input's error classes are removed as soon as the input is valid.
  • The scene's error classes are removed as soon as all its answers are valid and the user clicks next.
  • The most essential attribute is the data-field.

Demo


Ignoring Answer Validation

  • Answers don't have to be required.
  • The default behavior is to require all answers.
  • You can override this default with a data-required="false" on the anwer element.
  • You can mixed required and non-required answers on a scene.
  • Only the required answer and the scene will receive an error class
<input class="js-answer" data-field="required-test" data-required="false">
<input class="js-answer" data-field="required-test2">

Demo



Number Value Validation

  • Answers can be required to be a number and have a max/min value on it.
  • The max and min data attributes will automagically force the field to be a number.
  • The numeric data attribute will just ensure that the answer is a number.
  • An answer doesn't need both a max and a min.
<input class="js-answer" data-field="max-test" data-max="100">
<input class="js-answer" data-field="min-test" data-min="100">
<input class="js-answer" data-field="max-min-test" data-max="100" data-min="50">
<input class="js-answer" data-field="numeric-test" data-numeric="true">

Radio Element Answers

  • Answers can radios.
  • Storyboarder will not override any default radio behavior.
  • It assumes that you correctly structure your radios to reflect the toggle functionality you desire.
  • The key to successfully working with radios is where you put the data-field attribute.
  • You can break the storyboarder by making two radios that are are both required and tracked with different data-field attributes that are linked by default radio behavior.
<input type="radio" class="js-answer" data-field="radio-test" value="test1">
<input type="radio" class="js-answer" data-field="radio-test" value="test2">
<input type="radio" class="js-answer" data-field="radio-test-alt" value="test3">

Demo





Storyboarder Reporting

  • Answers are reported in a json-like object.
  • By default, answers are a single-dimensional value only.
  • The data-field attribute determines the key.
  • For checkboxes, radios, and selects, the value reported is the value attribute of the option selected.
  • For inputs, the value reported is the text within the field.
  • To print, use the Storyboarder.report mehtod.
